🎯 Darts Score Tracker
Простое приложение на C# Windows Forms для подсчёта счёта в игре в дартс по правилам 501 для двух игроков.

📋 Описание
Приложение позволяет двум игрокам (Красный и Зелёный) вести счёт в классической игре в дартс "501". 
Каждый игрок начинает с 501 очка и по очереди вводит результаты трёх бросков. Побеждает тот, кто первым точно обнулит свой счёт.

🎮 Правила игры (501)
Каждый игрок начинает с 501 очка
За ход игрок делает 3 броска, сумма которых вычитается из текущего счёта
Если после вычитания счёт становится ровно 0 — игрок побеждает 🏆
Если сумма бросков превышает остаток — происходит "перебор" (bust), ход аннулируется
Игра автоматически сбрасывается после победы или перебора

⚙️ Функционал

Для каждого игрока:

🔢 Три текстовых поля для ввода результатов бросков
🔘 Кнопка подтверждения хода
📊 Отображение текущего счёта
🔄 Счётчик сделанных ходов

Автоматически:

✅ Валидация введённых данных (только целые числа)
✅ Проверка на "перебор" (bust)
✅ Определение победителя
✅ Сброс игры после завершения раунда
✅ Очистка полей ввода после каждого хода

💻 Использование

Запустите приложение
Красный игрок вводит результаты трёх бросков в поля textBox1Red, textBox2Red, textBox3Red
Нажимает кнопку buttonRed для подтверждения хода
Приложение обновляет счёт и количество ходов
Повторите шаги 2–4 для Зелёного игрока
Игра продолжается, пока один из игроков не обнулит счёт точно
После победы или "перебора" игра автоматически сбрасывается
⚠️ Важно: Если сумма бросков превышает оставшиеся очки, ход не засчитывается, и счёт остаётся неизменным.

❓ FAQ
Q: Что делать, если игрок ввёл не число?
A: Приложение проигнорирует некорректный ввод — убедитесь, что все три поля заполнены целыми числами.
Q: Можно ли изменить стартовый счёт?
A: В текущей версии стартовый счёт фиксирован — 501. Для изменения потребуется правка кода в полях scoreRed и scoreGreen.
Q: Почему игра сбрасывается после перебора?
A: Это упрощённая логика. В классических правилах при переборе ход просто аннулируется, но счёт не сбрасывается. Вы можете доработать метод ResetGame() под свои нужды.
